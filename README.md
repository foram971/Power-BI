## 📊 WarmHands Retail Analytics Project (Power BI Case Study)

This project is built using a **real-world case study and multi-table Excel dataset** to analyze retail performance, profitability, inventory, and customer behaviour using Power BI.

---

## 🎯 Project Objective

The objective of this project is to analyze retail business data across multiple dimensions such as orders, customers, stock, pricing, cost, and geography to:

- Understand sales and profit trends  
- Identify key revenue-driving products  
- Analyze customer purchasing behaviour  
- Evaluate stock impact on sales performance  
- Support data-driven business decisions  

---

## 📂 Dataset Overview

The dataset consists of multiple Excel sheets:

- 📦 Orders – Transaction-level sales data  
- 👤 Customers – Customer details and identifiers  
- 📊 Stock – Inventory availability data  
- 💰 Price – Product pricing details  
- 📉 Cost – Cost structure per product  
- 🌍 Country – Geographic mapping data  

The dataset is relational in nature and was used to build a data model in Power BI.

---

## 🛠 Tools & Technologies

- Power BI Desktop  
- Power Query (Data Cleaning & Transformation)  
- DAX (Calculated Measures & KPIs)  
- Excel (Data Source)

---

## 🔄 Data Preparation & Modeling

- Imported multiple Excel sheets into Power BI  
- Cleaned and transformed data using Power Query  
- Standardized data types and removed inconsistencies  
- Created relationships between tables (Star Schema Model)  
- Built calculated columns and DAX measures for KPIs  
- Integrated all tables into a unified data model  

---

## 🧠 Case Study Approach

Each business question was solved step-by-step using Power BI:

- 📈 Sales performance analysis across products and time  
- 💹 Profitability analysis (Price vs Cost comparison)  
- 👥 Customer segmentation and behaviour analysis  
- 📦 Stock availability impact on sales  
- 🌍 Country-wise performance evaluation  

### 📊 ABC Analysis

To understand product contribution, I applied ABC Analysis:

- **Category A:** High-value products contributing major revenue  
- **Category B:** Medium-value stable products  
- **Category C:** Low-value products with minimal impact  

This helped identify:
- High-priority revenue-driving products  
- Optimization opportunities  
- Inventory focus areas  

---

## 📊 Dashboard Features

- KPI Cards (Total Sales, Profit, Cost, Orders)  
- Sales Trend Analysis  
- Profitability Breakdown by Product  
- Customer Analysis  
- Stock vs Sales Comparison  
- Country-wise Performance View  

---

## 🔍 Key Insights

- A small number of products generated most of the revenue (Pareto principle)  
- Some high-selling products had low profitability due to the cost structure  
- Stock availability directly impacted sales performance  
- Customer behaviour varied significantly across regions  
- Certain countries contributed disproportionately to revenue
